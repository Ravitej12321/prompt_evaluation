###Role: You are a DevOps engineer responsible for optimizing CI/CD pipelines for production-grade systems.

Context: The current CI/CD pipeline is inefficient, slow, and lacks proper security practices. Builds take too long, tests are unreliable, and deployments are prone to failure.

Task: Analyze the existing CI/CD pipeline and identify performance bottlenecks, reliability issues, and security vulnerabilities.

Requirements:

Pipeline Analysis:
- Review all stages: build, test, and deployment
- Identify redundant or slow steps
- Detect lack of parallelization opportunities

Performance Issues:
- Analyze build times and suggest improvements
- Identify inefficient dependency management
- Recommend caching strategies

Reliability Issues:
- Detect flaky or inconsistent test behavior
- Identify missing failure handling or retry mechanisms
- Evaluate deployment stability

Security Gaps:
- Identify hardcoded secrets or exposed credentials
- Check for insecure artifact handling
- Suggest best practices for secrets management

Output:
- Detailed report of issues categorized into performance, reliability, and security
- Suggested improvements with reasoning
- Example CI/CD configuration snippets (e.g., GitHub Actions, GitLab CI)
- Priority-based action plan (high → low impact)

Constraints:
- Solutions must be scalable for large teams
- Must follow industry DevOps best practices
- Avoid introducing unnecessary complexity
